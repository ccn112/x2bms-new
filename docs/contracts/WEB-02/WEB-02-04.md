# WEB-02-04 — Duyệt cư dân

- **Source image:** `handoff/.../WEB-02_Cư_dân_and_Căn_hộ/images/WEB-02-04__Duyệt_cư_dân.png`
- **Type:** approval list (with working decisions)
- **Render strategy:** Custom Page — `App\Livewire\ResidentApprovalQueue` (`GET /resident-approvals`). Livewire actions `approve/reject/needMore`.
- **X2 components:** X2AdminShell, X2ActionBar, X2KpiCard ×4, X2SectionCard, X2StatusBadge; inline match-score progress bars.
- **Canonical entities/tables:** `resident_approval_requests`, `apartments`, `residents`, `resident_apartment_relations`, `audit_logs`.
- **Data source (no hardcode):** KPIs pending/avg match/approved/need-more. Cards = pending requests (applicant, requested role, apartment, match score, docs, submitted).
- **Workflow:** `approve()` creates a `Resident` + `resident_apartment_relation` and writes an `audit_logs` row (action `resident.approve`); `reject()`/`needMore()` update status + audit. Decisions re-render the queue live.
- **Sample data shown:** Hồ sơ chờ duyệt, Độ khớp dữ liệu, Căn hộ đề nghị, Giấy tờ, Lịch sử.
- **Seed:** 8 pending applicants (match 54–95%, 1–5 docs).
- **RBAC/tenancy:** auth + `BelongsToTenant`; future permission `resident.approve` (mandated audit per ARCHITECTURE).
- **Acceptance:** `/resident-approvals` shows queue; clicking **Duyệt** removes the card, creates resident + relation + audit log.
- **Screenshot test:** `tests/Browser/web-02.spec.ts` → `WEB-02-04 resident approval queue + approve workflow` (asserts a card disappears after approve).
